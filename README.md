# Semantic Segmentation Evaluation: Cityscapes & SydneyScapes

> **Note:** The code in this repository was generated by Gemini for coding assistance.

This repository contains evaluation scripts and notebooks for various semantic segmentation models, specifically tailored for the **Cityscapes** and **SydneyScapes** datasets.

## Project Structure

- `Cityscapes/`: Contains evaluation codes and results for the Cityscapes dataset.
- `SydneyScapes/`: Contains evaluation codes and results for the SydneyScapes dataset.
- `requirements.txt`: List of required Python libraries.

## Supported Models

The following models are supported for evaluation:
- **Mask2Former**
- **MaskFormer**
- **PIDNet**
- **SAM2 (Segment Anything Model 2)**
- **Segformer**

## Setup Instructions

### 1. General Requirements
Install the necessary Python libraries using pip:
```bash
pip install -r requirements.txt
```

### 2. Special Instructions for PIDNet
To run the PIDNet evaluation scripts (`eva_pid.py` or `eva_PIDNet.py`), additional setup is required:

- **Clone the Official Repository:**
  You need the model definition from the official PIDNet implementation. Clone the following repository into your local environment:
  [GitHub: XuJiacong/PIDNet](https://github.com/XuJiacong/PIDNet)
  
  Ensure the `models` directory from the PIDNet repository is accessible in your Python path or placed in the same directory as the evaluation scripts.

- **Download Pre-trained Weights:**
  Download the appropriate pre-trained weights (e.g., `PIDNet_L_Cityscapes_test.pt`) from public sources like [kaggle](https://www.kaggle.com/datasets/sargamgoya1/pidnet-s-weights) or the official links provided in the PIDNet repository.

## How to Use

### Using Jupyter Notebooks
Most models (Mask2Former, MaskFormer, SAM2, Segformer) have their evaluation logic in `.ipynb` files. 
1. Navigate to the `evaluation_codes/` directory of the desired dataset.
2. Open the notebook (e.g., `eva_segformer.ipynb`).
3. Follow the steps within the notebook to run the evaluation.

### Running Python Scripts (PIDNet)
The PIDNet evaluation is provided as standalone Python scripts. Run them from the terminal using the following format:

**For SydneyScapes:**
```bash
python SydneyScapes/evaluation_codes/eva_pid.py \
    --model s \
    --weight /path/to/pidnet_s.pt \
    --img_dir ./SydneyScapes/leftImg8bit/test/NSWday \
    --gt_dir ./SydneyScapes/gtFine/test/NSWday \
    --num_classes 19
```

**For Cityscapes:**
```bash
python Cityscapes/evaluation_codes/eva_PIDNet.py \
    --model l \
    --weight /path/to/pidnet_l.pt \
    --img_dir ./Cityscapes/leftImg8bit/val \
    --gt_dir ./Cityscapes/gtFine/val \
    --num_classes 19 \
    --eval_scale 0.5
```

#### Detailed Arguments for PIDNet:
- `--model`: Model size variant (`s`, `m`, or `l`). (Default: `s`)
- `--weight`: **(Required)** Path to the pre-trained `.pt` checkpoint file (e.g., from [Hugging Face](https://huggingface.co/models?search=pidnet)).
- `--img_dir`: **(Required)** Path to the source image directory (e.g., `leftImg8bit`).
- `--gt_dir`: **(Required)** Path to the ground truth annotation directory (e.g., `gtFine`).
- `--num_classes`: Number of semantic classes in the dataset (Default: `19`).
- `--eval_scale`: *(Cityscapes script only)* Resizes images during evaluation to save memory/time. `1.0` is full size, `0.5` is half size. (Default: `0.5`)

## Results
Evaluation reports, including mIoU metrics and sample predictions, are saved in the respective `evaluation_results_*` directories. Carbon emission data is also tracked using `codecarbon` and saved as `emissions.csv`.
